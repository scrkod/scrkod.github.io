О языке.

C# (CSharp) — объектно-ориентированный язык программирования. Относится к семье языков с C-подобным синтаксисом, из них его синтаксис наиболее близок к C++ и Java. C# – это многопарадигмальный язык программирования, который широко используется не только на платформе Windows, но и на платформе iOS/Android (благодаря Xamarian и Swift) и даже на Linux.

Язык был выпущен компанией Microsoft вместе с платформой ".NET". C# и .NET в некотором смысле являются "родственными душами": некоторые свойства языка специально разрабатывались, чтобы обеспечить комфортную работу в среде .NET, в то же время и некоторые свойства .NET специально закладывались для поддержки C#, хотя .NET нацелена на использование многих языков программирования. Компилятор с C# входит в стандартную установку самой .NET, поэтому программы на нём можно создавать и компилировать даже без инструментальных средств, вроде Visual Studio.

История впопыхах.

Язык программирования С# был разработан в 1993—2001 годах группой инженеров компании Microsoft под руководством Андерса Хейлсберга и Скотта Вильтаумота как язык разработки приложений для платформы Microsoft .NET и впоследствии был стандартизирован как ECMA-334 и ISO/IEC 23270. Тестовая версия вышла в 2000 году, а окончательная — в 2002-м. Вместе с ней появился Microsoft Visual Studio — новый удобный редактор кода. Нужно подметить, что язык C# развивается параллельно платформе Microsoft .NET (и Mircosoft Visual Studio), поэтому зачастую правильно говорить, что определённой версии языка соответствует определённая версия платформы .NET.

Веб-разработка и фреймворки.

ASP.NET

Документация: https://learn.microsoft.com/ru-ru/aspnet/core/whats-new/?view=aspnetcore-8.0

ASP.NET – это часть технологии .NET, используемая для написания мощных клиент-серверных интернет приложений. Она позволяет создаватьдинамические страницы HTML. ASP.NET возникла в результате объединения более старой технологии ASP (активные серверные страницы) и .NET Framework. Она содержит множество готовых элементов управления, используя которые можно быстро создавать интерактивные образовательные веб-сайты.

Обычно приложение ASP.NET можно разбить на набор компонентов:

  • веб-формы (страницы .aspx). Обеспечивают пользовательский интерфейс Web-приложения;
  • страницы code-behind - программный код для веб-форм;
  • файлы конфигурации. Для каждого веб-приложения предусмотрен файл Web.config, для всего веб-сервера - machine.config.

При создании нового проекта ASP.NET нам предлагается выбрать язык по умолчанию для создания страниц codebehind: Visual Basic, C# или J#. С точки зрения конечного результата разницы нет практически никакой, так как функциональные возможности любого языка практически идентичны и все они работают с единой библиотекой .NET Framework.

Работа по созданию приложения ASP.NET складывается из следующих шагов:

  1. в режиме Дизайнера в графическом режиме создается содержимое страницы, на ней размещается текст, изображения, другие элементы оформления (например, таблицы);
  2. в режиме Дизайнера на страницу из ToolBox перетаскиваются элементы управления (текстовые поля, кнопки, ниспадающие списки и т.п.);
  3. на странице codebehind для событий самой страницы и элементов управления создается программный код;
  4. при необходимости на вкладке Source дорабатывается код HTML для страницы.

Платформа ASP.NET позволяет более удобно управлять необходимыми модулями для веб-приложения, с помощью их загрузки через пакетный менеджер NuGet. Также при разработке веб-приложения нет строгой зависимости от библиотеки System.Web.dll. Платформа включает в себя фреймворк MVC, в котором реализуются функциональности MVC, Web Pages и Web API. В связи с этим исключается большая часть повторяющихся функциональностей, они соединены в одну модель – ASP.NET MVC.
